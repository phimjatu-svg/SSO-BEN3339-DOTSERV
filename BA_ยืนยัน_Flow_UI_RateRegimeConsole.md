## 1. ภาพรวมระบบ

ระบบแบ่งเป็น 2 ส่วนงาน (สะท้อนเป็น 2 กลุ่มแท็บใน UI):

1. **จัดการประกาศ (ภายใน)** — เจ้าหน้าที่กำหนดอัตรา/เพดาน/วันมีผล เป็น config (single source of truth) ผ่าน workflow maker–checker · 3 หน้า: ทะเบียน/ฟอร์ม · อนุมัติประกาศ · สถานะการทำรายการ
2. **วินิจฉัยผู้ประกันตน** — ใช้ข้อมูลผู้ประกันตนจริง คำนวณสิทธิ โดยดึงอัตรา/เพดานจากประกาศข้างต้น · 1 หน้า: วินิจฉัยสิทธิว่างงาน (รองรับหลายรับแจ้ง + ตรวจซ้ำซ้อน + Timeline)

**บทบาท & สิทธิ์ (ต้องยืนยัน):**

| บทบาท | ระดับ | สิทธิ์ |
|---|---|---|
| ผู้เสนอ (Maker) | ปฏิบัติการ | สร้าง/แก้/ลบ ประกาศ · เสนออนุมัติ — **อนุมัติไม่ได้** |
| ผู้อนุมัติ (Checker) | **ชำนาญการ** | อนุมัติ / ตีกลับ / ยกเลิก (maker ≠ checker) |
| จนท.วินิจฉัย | ปฏิบัติการ+ | ใช้หน้าวินิจฉัย (อ่านอัตราจากประกาศที่มีผล) |

## 2. Flow A — จัดการประกาศ (maker–checker)

แผนผัง: `flow_จัดการประกาศ_makerchecker.mermaid`

ลำดับขั้น:
1. ผู้เสนอ (ปฏิบัติการ) ที่หน้า **ทะเบียน/ฟอร์ม** กด "สร้างประกาศ" หรือคลิกชื่อประกาศ → "ขอแก้ไข/ขอลบ"
2. กรอกข้อมูล → ระบบ validate (ห้ามช่วงวันมีผลทับซ้อน · ห้ามลบ master · ช่วงว่างใช้ master 2547)
3. กด "เสนออนุมัติ" → สร้างรายการสถานะ **รออนุมัติ** → ระบบพาไปหน้า **อนุมัติประกาศ**
4. ที่หน้า **อนุมัติประกาศ** — ถ้าผู้ใช้เป็น *ปฏิบัติการ* ปุ่มอนุมัติ/ตีกลับ **ถูกล็อก**; ต้องเป็น *ชำนาญการ* จึงกดได้
5. ผู้อนุมัติ (ชำนาญการ) ตรวจ diff/validation → **อนุมัติ** (ประกาศมีผล + bump version) / **ตีกลับ** / **ยกเลิก**
6. ทุกสถานะติดตามได้ที่หน้า **สถานะการทำรายการ** (stepper + audit + diff); ตีกลับ → "แก้ไข & ส่งใหม่"

### ตาราง ขั้นตอน ↔ หน้าจอ ↔ สิ่งที่ BA ยืนยัน

| # | ขั้นตอน | หน้าจอ (prototype) | สิ่งที่ BA ต้องยืนยัน | ✔ / ✘ / หมายเหตุ |
|---|---|---|---|---|
| A1 | สร้าง/แก้/ลบ ประกาศ | ทะเบียน/ฟอร์ม → modal | field ประกาศครบ (อัตรา/เพดานวัน/เพดานค่าจ้าง/วันมีผล/อ้างอิงกฎหมาย) |  |
| A2 | Validation | ฟอร์ม (hint สีเหลือง) | กฎ: ห้ามทับซ้อน · gap = master · ห้ามลบ master · effTo ≥ effFrom |  |
| A3 | เสนออนุมัติ | ปุ่ม "เสนออนุมัติ" | สร้างคำขอสถานะ "รออนุมัติ" เข้าคิว |  |
| A4 | สิทธิอนุมัติตามระดับ | อนุมัติประกาศ (ตัวสลับสิทธิ) | **อนุมัติได้เฉพาะระดับชำนาญการ** · ปฏิบัติการถูกล็อก |  |
| A5 | อนุมัติ/ตีกลับ/ยกเลิก | อนุมัติประกาศ | ผลของแต่ละปุ่ม (create→มีผล · edit→อัปเดต+version · delete→ยกเลิก) |  |
| A6 | ติดตามสถานะ | สถานะการทำรายการ | stepper 4 ขั้น · diff · ข้อมูลประกาศ · ปุ่มแก้ไข&ส่งใหม่ |  |
| A7 | สถานะครอบคลุม | ทุกหน้า (badge) | ร่าง · รออนุมัติ · ตรวจสอบ · มีผลแล้ว · ตีกลับ · ยกเลิก |  |
| A8 | บันทึก/เก็บประวัติ | (effective-dated) | เก็บทุกเวอร์ชัน · ประกาศที่ใช้วินิจฉัยแล้ว = immutable |  |

## 3. Flow B — วินิจฉัยผู้ประกันตน

แผนผัง: `flow_วินิจฉัยผู้ประกันตน.mermaid`

ลำดับขั้น:
1. ที่หน้า **วินิจฉัยสิทธิว่างงาน** เลือกจาก **คลังเทสเคส** (เดโม) หรือกรอกข้อมูลผู้ประกันตนจริง — รองรับ **หลายรับแจ้ง (คำขอ A/B…)**
2. ระบบ **ดึงอัตรา/เพดานจาก regime config** ตามวันเกิดสิทธิ + ประเภทการว่างงาน (ถ้าไม่มีประกาศเฉพาะ → master 2547)
3. คำนวณ: ค่าจ้างรายวัน (ม.57) · รับวันละ % · **สิทธิเต็มล็อกตามประกาศ** · **ค่าจ้างล็อกเพดาน** · จ่ายจริง · โควตา/ปีปฏิทิน
4. แสดง **ตารางงวด** (จ่าย 2 รอบ + auto-fill เดือนขาด + เรียกคืน) และ **Timeline** เทียบทุกรับแจ้ง
5. ตรวจ **ความซ้ำซ้อน**: ถ้าช่วงจ่ายจริงทับกัน → ไฮไลต์แดง + ยอดเรียกคืน; ถ้าคร่อมขอบประกาศ → ปิดรับแจ้ง #1/เปิด #2
6. สรุปผลวินิจฉัย — จ่ายรวม (วัน/บาท)

### ตาราง ขั้นตอน ↔ หน้าจอ ↔ สิ่งที่ BA ยืนยัน

| # | ขั้นตอน | หน้าจอ (prototype) | สิ่งที่ BA ต้องยืนยัน | ✔ / ✘ / หมายเหตุ |
|---|---|---|---|---|
| B1 | หลายรับแจ้ง | วินิจฉัย → "เพิ่มคำขอ" | เพิ่ม/ลบ คำขอ A/B/C ได้ |  |
| B2 | field วินิจฉัย | คำขอ (ฟอร์ม) | ครบ: ประเภท · ค่าจ้าง 3 เดือน · วันลาออก · กลับเข้าทำงาน · นายจ้างแจ้ง · ขึ้นทะเบียน · เกิดสิทธิ(auto) · สิทธิเต็ม |  |
| B3 | ดึงอัตราจากประกาศ | banner "ใช้ประกาศ:" | อัตรา/เพดานมาจาก regime ที่มีผล ณ วันเกิดสิทธิ (fallback master) |  |
| B4 | ล็อกตามประกาศ | field สิทธิเต็ม / ค่าจ้าง | สิทธิเต็ม = เพดานวัน (ล็อก) · ค่าจ้างไม่เกินเพดาน (ล็อก) |  |
| B5 | รายงานตัว/ตารางงวด | "ดูตารางงวด" | จ่าย 2 รอบ · auto-fill เดือนขาด · ตัดโควตา · เรียกคืน |  |
| B6 | Timeline + ซ้ำซ้อน | Timeline + legend | ไฮไลต์ช่วงซ้ำซ้อนสีแดง · หมุดขึ้นทะเบียน/รายงานตัว · ยอดเรียกคืน |  |
| B7 | คร่อมสิทธิ | เทสเคส "คร่อมสิทธิโควิด" | ปิดรับแจ้ง #1 ที่ขอบ / เปิด #2 ต่อเนื่อง (ไม่ถือซ้ำซ้อน) |  |
| B8 | คลังเทสเคส | dropdown คลังเทสเคส | เลือก scenario auto-fill · บันทึกเทสเคสเองได้ |  |
| B9 | สรุปผล | การ์ดสรุปผลวินิจฉัย | จำนวนคำขอ · ซ้ำซ้อน(วัน) · โควตา · เรียกคืน · จ่ายรวม |  |

## 4. จุดที่ต้องตัดสินใจ / ยืนยันเชิงนโยบาย (BA decision)

| # | ประเด็น | ค่าที่ระบบใช้ตอนนี้ | ต้องการการยืนยัน |
|---|---|---|---|
| D1 | สิทธิอนุมัติ | เฉพาะระดับชำนาญการ | ระดับขั้นต่ำที่อนุมัติได้ถูกต้องหรือไม่ |
